# Region 角色 - 需求分析与用例规划

> 页面路径：平台管理 → Region 管理 → Region 角色
> 相关模块：Region 成员、Region 用户、Region 用户组、权限管理

---

## 一、模块划分

| 模块编号 | 模块名称 | 说明 |
|---------|---------|------|
| M1 | Region 角色列表页 | 列表展示、搜索、刷新、列设置、分页、行内操作 |
| M2 | 创建 Region 角色 | 创建弹窗、名称/别名/描述字段校验、进入权限编辑 |
| M3 | 编辑角色信息 | 别名与描述编辑、名称只读 |
| M4 | 编辑角色权限 | 分类权限选择、依赖/互斥、保存回显 |
| M5 | 删除 Region 角色 | 二次确认、预置角色保护、关联用户/用户组策略 |
| M6 | Region 角色详情页 | 基本信息、权限 Tab、授权用户 Tab、授权用户组 Tab |
| M7 | 权限与兼容性 | 不同角色访问权限、国际化、浏览器兼容性 |

## 二、核心规则提炼

### 2.1 角色属性规则

| 字段 | 必填 | 格式规则 | 长度限制 |
|------|------|---------|---------|
| 名称 | 是 | 只能包含小写字母、数字和连字符（-），必须以小写字母或数字开头和结尾 | 最长 63 个字符 |
| 别名 | 否 | 任意字符 | 最长 63 个字符 |
| 描述 | 否 | 任意字符 | 最长 256 个字符 |

### 2.2 预置角色

- `region-admin`：Region 管理员
- `region-viewer`：Region 只读用户
- `region-regular`：Region 普通用户

预置角色通常不允许删除或修改名称，权限可能受保护。

### 2.3 权限分类

| 分类 | 示例权限 |
|------|---------|
| 业务空间 | 业务空间创建、业务空间管理、业务空间查看 |
| 集群管理 | 集群查看、集群管理 |
| 访问控制 | Region 用户查看、Region 用户组查看、Region 角色查看、Region 用户管理、Region 用户组管理、Region 角色管理 |

## 四、待确认项

1. 预置角色是否允许删除？是否允许编辑权限？
2. 角色已授权用户/用户组时，删除是强制解绑还是禁止删除？
3. 权限是否存在依赖或互斥规则（如「管理」是否自动包含「查看」）？
4. 列表每页默认显示条数是多少？
5. 搜索框支持按名称还是按别名/描述搜索？
6. 详情页授权用户/用户组列表是否支持批量操作？
7. 是否支持通过 YAML 查看或编辑角色？
