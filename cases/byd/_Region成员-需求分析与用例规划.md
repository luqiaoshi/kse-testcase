# Region 成员 - 需求分析与用例规划

> 页面路径：平台管理 → Region 管理 → Region 成员  
> 页面标题：Region 成员  
> 页面 Tab：用户（默认）、用户组

## 二、核心规则提炼

### 2.1 页面基础规则

- 默认进入「用户」Tab，展示当前 Region 下已邀请的用户成员。
- 用户列表支持按用户名/邮箱模糊搜索。
- 用户列表支持复选框单行选择、全选及批量操作。
- 用户列表支持刷新、自定义显示字段、分页。
- 切换至「用户组」Tab，展示当前 Region 下已邀请的用户组。
- 用户组列表为空时显示空状态提示及邀请入口。

### 2.2 邀请成员规则

- 邀请用户：点击「邀请」打开弹窗，搜索用户名，选择用户后分配角色（region-viewer / region-regular / region-admin）。
- 邀请用户组：同用户邀请逻辑，搜索用户组名称，选择后分配角色。
- 已选用户/用户组显示已分配角色及对勾标识。
- 支持一次邀请多个用户/用户组，但每个用户/用户组只能被分配一个角色。
- 已被邀请的用户/用户组不应再次出现在可邀请列表中。

### 2.3 角色规则

- 可选角色：
  - `region-viewer`：查看 region 中的所有资源。
  - `region-regular`：被邀请加入集群或业务空间之前无法访问任何资源。
  - `region-admin`：管理 region 中的所有资源。
- 修改角色时角色字段必填，下拉选择。
- 用户组角色修改逻辑与用户一致。

### 2.4 移除成员规则

- 移除用户：需输入正确的用户名以确认风险操作。
- 移除用户组：需输入正确的用户组名以确认风险操作。
- 输入不匹配时「确定」按钮禁用或点击后提示错误。
- 支持批量移除，批量移除通常为通用二次确认（需按产品设计确认是否逐条输入名称）。
- 移除后列表同步刷新，被移除成员不再显示。

### 2.5 用户组详情规则

- 点击用户组名称进入详情页。
- 详情页展示用户组基本信息、描述、操作下拉、属性面板（Region 角色、加入时间）。
- 详情页包含「用户」Tab，展示该用户组下成员列表。
- 详情页支持返回用户组列表。
