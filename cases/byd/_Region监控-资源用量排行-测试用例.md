# Region 监控 - 资源用量排行 - 测试用例

> 本用例针对 Region 监控下「资源用量排行」Tab 页签进行细化，覆盖维度切换、指标切换、排序、搜索、刷新、分页、空态、字段校验及国际化/权限等场景。

---

## 模块一：页面加载与默认展示

### tc-P0-功能测试：正常进入资源用量排行页签，默认展示集群维度排行列表

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且集群有实际资源使用数据。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「资源用量排行」
2. 等待页面列表加载完成
3. 查看默认选中的维度 Tab
4. 查看默认排行指标
5. 查看列表字段与数据

#### 预期结果
- 页面正常加载，无白屏、无长时间空白
- 列表数据加载成功，无失败或长时间空白
- 默认选中「集群」维度 Tab
- 默认按 CPU 用量（或设计默认指标）排行展示集群列表
- 列表字段完整，数据正确显示，无 `--` 或空值（除非实际为空）
  1. 默认排序方向为降序（以实际设计为准）

### tc-P1-功能测试：资源用量排行默认指标与单位正确

#### 前置条件 (pc)
- 已进入 Region 监控「资源用量排行」页签。
- 默认展示集群维度列表。

#### 测试步骤
1. 查看默认指标选择器展示内容
2. 查看列表中数值单位
3. 查看指标下拉列表可选项（如 CPU 用量、内存用量、磁盘用量等）
4. 对比各选项与设计说明

#### 预期结果
- 默认指标正确（如 CPU 用量）
- 列表中数值单位与指标对应（如 CPU 用量为核数或 %，内存为 GB 等）
- 指标下拉列表包含 CPU 用量、内存用量、磁盘用量等可选项（以实际支持为准）
- 各指标名称文案正确，无缺失

## 模块二：维度切换

### tc-P0-功能测试：切换集群/节点/命名空间/容器组 Tab，列表列头和数据正确切换

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且各维度均有实际资源使用数据。
- 已进入 Region 监控「资源用量排行」页签。

#### 测试步骤
1. 记录当前「集群」维度下列表列头和数据
2. 点击「节点」Tab
3. 查看列表列头和数据变化
4. 点击「命名空间」Tab
5. 点击「容器组」Tab
6. 依次切回「集群」「节点」「命名空间」「容器组」，观察数据一致性

#### 预期结果
- 「集群」维度下列表列头和数据记录正确
- 「节点」Tab 下列表列头切换为节点维度字段（如节点名称、CPU 用量、内存用量等）
- 列表列头和数据变化正确，各维度下数据与后端接口返回一致，数据范围按当前 Region/集群过滤
- 「命名空间」Tab 下列表列头切换为命名空间维度字段
- 「容器组」Tab 下列表列头切换为容器组维度字段
- 依次切回各维度后，搜索条件、排序状态、分页等按设计重置或保留（以实际设计为准），无数据错乱

### tc-P1-功能测试：各维度下表格字段完整且数值正确

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群。
- 已准备或可通过后端接口查询各维度资源用量数据。
- 已进入 Region 监控「资源用量排行」页签。

#### 测试步骤
1. 进入「集群」维度，核对列头与各行数据
2. 进入「节点」维度，核对列头与各行数据
3. 进入「命名空间」维度，核对列头与各行数据
4. 进入「容器组」维度，核对列头与各行数据
5. 对比各维度关键字段数值与后端接口数据

#### 预期结果
- 「集群」维度字段完整，各集群 CPU/内存/磁盘等指标数值正确
- 「节点」维度字段完整，各节点名称、IP、CPU/内存等指标数值正确
- 「命名空间」维度字段完整，各命名空间资源用量数值正确
- 「容器组」维度字段完整，容器组名称、所属命名空间/节点、资源用量数值正确
- 各维度数值与后端接口返回数据一致，无计算错误或显示异常

### tc-P2-安全性测试：无权限用户访问资源用量排行控制

#### 前置条件 (pc)
- 已使用非 platform-admin 角色登录（如 platform-regular、自定义只读角色）。
- 平台下已存在 Region 及成员集群。

#### 测试步骤
1. 使用无权限用户登录平台
2. 直接访问资源用量排行页 URL
3. 尝试调用资源用量排行相关 API 接口
4. 切换为 platform-admin 角色，再次访问并对比

#### 预期结果
- 无权限用户无法通过 UI 进入资源用量排行页，或页面提示无权限
- 直接访问 URL 时，页面跳转至无权限提示或 403 页面
- 相关 API 接口返回 403 无权限
- platform-admin 角色可正常访问并查看全部数据；数据权限隔离正确，不会越权展示其他 Region/集群数据
