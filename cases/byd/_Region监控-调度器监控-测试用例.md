# Region 监控 - 调度器监控 - 测试用例

> 本用例针对 Region 监控下「调度器监控」Tab 页签进行细化，覆盖调度次数、调度速率、调度延迟图表加载、分类展示、分位线展示、集群/时间切换、异常/空态及国际化/权限等场景。

---

## 模块一：页面加载与图表展示

### tc-P0-功能测试：正常进入调度器监控页签，调度相关图表加载成功

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且调度器有实际调度数据。
- 监控组件（Prometheus/Whizard）正常运行。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「调度器监控」
2. 等待页面全部图表加载完成
3. 查看调度次数图表
4. 查看调度速率(次/秒)图表
5. 查看调度延迟(ms)图表
6. 对比各图表数据与后端监控接口返回数据

#### 预期结果
- 页面正常加载，无白屏、无长时间空白
- 页面顶部集群选择器和时间范围选择器默认显示正确
  调度次数、调度速率、调度延迟三个图表均正常渲染
- 调度次数图表标题、单位、坐标轴、图例完整显示
- 调度速率(次/秒)图表标题、单位、坐标轴、图例完整显示
- 调度延迟(ms)图表标题、单位、坐标轴、图例完整显示
- 各图表数据与后端接口返回一致，无 `--` 或空值（除非无数据）
  图表布局整齐，无错位、重叠或截断

### tc-P1-功能测试：调度器监控图表单位与标题正确

#### 前置条件 (pc)
- 已进入 Region 监控「调度器监控」页签。
- 页面已有调度监控数据。

#### 测试步骤
1. 查看调度次数图表标题与 Y 轴单位
2. 查看调度速率图表标题与单位
3. 查看调度延迟图表标题与单位
4. 查看各图表图例文案

#### 预期结果
- 标题正确，单位为 `次` 或按设计展示
- 标题为「调度速率」，单位为 `次/秒`
- 标题为「调度延迟」，单位为 `ms`
- 各图表图例与对应分类一致，颜色区分明显

## 模块四：集群与时间范围切换

### tc-P0-功能测试：切换集群后调度器图表数据刷新

#### 前置条件 (pc)
- 平台下已存在包含多个成员集群的 Region。
- 各集群调度器数据存在明显差异。
- 已进入 Region 监控「调度器监控」页签。

#### 测试步骤
1. 记录当前集群选择器和各图表数据
2. 在页面顶部集群选择器中切换至另一个成员集群
3. 观察调度次数、调度速率、调度延迟图表加载与刷新过程
4. 记录切换后各图表数据并与切换前对比

#### 预期结果
- 基线数据（当前集群选择器、各图表数据）完整可用
- 切换后页面给出加载提示，所有图表按新集群重新加载
- 调度次数、调度速率、调度延迟图表数据均刷新为所选集群的数据
  成功/错误/失败分类和 99th/90th/50th/平均值分位线均按新集群刷新
- 切换后的数据与所选集群实际调度器监控数据一致
  无旧数据残留、无数据错乱；切换过程中页面不白屏、不报错

## 模块五：异常与空态

### tc-P1-功能测试：无调度数据或接口异常时的空态/错误提示

#### 前置条件 (pc)
- 当前 Region/集群下无调度监控数据。
- 或通过 Mock 模拟调度器监控接口返回空数据或异常。
- 已进入 Region 监控「调度器监控」页签。

#### 测试步骤
1. 进入调度器监控页签
2. 查看各图表空态展示
3. 模拟调度次数接口返回空数据，刷新页面
4. 模拟调度延迟接口 500 错误，刷新页面
5. 模拟全部接口异常，刷新页面
6. 恢复接口后刷新页面

#### 预期结果
- 无调度数据时各图表展示空态提示（如「暂无数据」或空折线图）
- 空态/错误提示文案正确，符合当前语言环境
- 调度次数接口返回空数据时，对应图表展示空态提示（如「暂无数据」或空折线图），其他图表正常加载
- 调度延迟接口 500 错误时，对应图表展示错误提示或占位，其他图表正常加载
- 全部接口异常时，页面展示明确的错误提示或重试入口，不白屏
  接口超时时，展示加载超时提示，不长时间卡死
- 恢复接口后，页面可正常刷新并展示数据

## 模块六：国际化与权限

#### 前置条件 (pc)
- 平台支持中英文语言切换。
- 已进入 Region 监控「调度器监控」页签，且有实际数据。

#### 测试步骤
1. 切换系统语言为中文
2. 查看页面标题、图表标题、图例、单位、提示信息等文案
3. 切换系统语言为英文
4. 查看上述文案的英文展示

#### 预期结果
- 中文环境下，所有文案均为中文，无截断或溢出
- 英文环境下，所有文案正确切换为英文，语义准确
- 图表标题、图例（成功/错误/失败、99th/90th/50th/平均值）、单位、空态/错误提示等均正确翻译
- 无未翻译的英文残留或乱码；切换语言后，图表数据展示不受影响

### tc-P2-安全性测试：无权限用户访问调度器监控控制

#### 前置条件 (pc)
- 已使用非 platform-admin 角色登录（如 platform-regular、自定义只读角色）。
- 平台下已存在 Region 及成员集群。

#### 测试步骤
1. 使用无权限用户登录平台
2. 直接访问调度器监控页 URL
3. 尝试调用调度器监控相关 API 接口
4. 切换为 platform-admin 角色，再次访问并对比

#### 预期结果
- 无权限用户无法通过 UI 进入调度器监控页，或页面提示无权限
- 直接访问 URL 时，页面跳转至无权限提示或 403 页面
- 接口返回 403 无权限
- platform-admin 角色可正常访问并查看全部数据；数据权限隔离正确，不会越权展示其他 Region/集群数据
