# Region 监控 - 概览 - 测试用例

> 本用例针对 Region 监控下「概览」Tab 页签的各子模块进行细化，覆盖全局资源统计、集群列表、核心监控指标、全局告警、容器组状态与 QoS、页面整体与交互等场景。

---

## 模块一：全局资源统计

### tc-P0-功能测试：全局资源统计卡片正常加载与数值正确

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且有实际资源数据。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「概览」
2. 查看页面顶部全局资源统计区域
3. 对比各统计卡片数值与 Region 实际资源数量

#### 预期结果
- 概览页加载成功，无白屏或加载失败提示
- 统计卡片依次展示：集群、节点、命名空间、容器组、部署、有状态副本集、守护进程集、任务、定时任务、持久卷声明、服务、应用路由；各卡片标题、图标、数值均正确显示
- 各卡片数值与后端接口返回数据一致，无 `--` 或空值（除非实际资源为空）

### tc-P1-功能测试：全局资源统计卡片数值与后端数据一致

#### 前置条件 (pc)
- 已准备或已知当前 Region 下各类资源的实际数量（可通过 kubectl/API 查询）。
- 已进入 Region 监控概览页。

#### 测试步骤
1. 通过后端 API 或 kubectl 获取当前 Region 下各类资源数量
2. 记录概览页全局资源统计区域各卡片数值
3. 分别对比集群、节点、命名空间、容器组、部署、服务、应用路由等卡片数值

#### 预期结果
- 后端 API 返回的资源总数与页面对应卡片数值一致
- 页面刷新后，卡片数值与后端实时数据保持一致
- 资源数量发生变化（如新增/删除 Pod）后，卡片数值在一定时间内自动刷新并同步

### tc-P2-功能测试：点击全局资源统计卡片跳转对应资源列表

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且有实际资源数据。
- 已进入 Region 监控概览页。

#### 测试步骤
1. 依次点击「容器组」「部署」「服务」「应用路由」等支持跳转的统计卡片
2. 查看跳转后的资源列表页数据范围
3. 点击浏览器返回按钮

#### 预期结果
- 点击支持跳转的卡片后，页面正常导航至对应资源列表页
- 跳转后的资源列表页默认按当前 Region/集群范围过滤，展示数据正确；不支持跳转的卡片点击后无异常行为
- 浏览器返回后仍停留在 Region 监控概览页

### tc-P2-功能测试：后端返回空数据时全局资源统计展示

#### 前置条件 (pc)
- 当前 Region 下某类资源（如定时任务、守护进程集）数量为 0。
- 或已通过 Mock/接口模拟返回空数据。

#### 测试步骤
1. 进入 Region 监控概览页
2. 查看全局资源统计区域中数量为 0 的卡片
3. 模拟后端接口返回空数组/null，刷新页面
4. 查看所有统计卡片的展示状态

#### 预期结果
- 数量为 0 的卡片显示为 0，不显示空值或 `--`
- 卡片布局正常，无错位或缺失
- 页面无报错、白屏或加载失败提示
- 空数据卡片不影响其他卡片的正常展示
