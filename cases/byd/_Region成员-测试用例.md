# Region 成员 - 测试用例

> 页面路径：平台管理 → Region 管理 → Region 成员  
> 页面标题：Region 成员  
> 页面 Tab：用户（默认）、用户组  
> 本用例覆盖 Region 成员页面「用户」与「用户组」两个 Tab 的列表展示、邀请、修改角色、移除、详情查看、权限及国际化等场景。

---

## 模块一：用户 Tab - 列表展示

### tc-P0-功能测试：用户 Tab 默认进入及列表字段完整

#### 前置条件 (pc)
- 已使用 platform-admin 角色登录平台。
- 当前 Region 下已存在至少一个用户成员，例如 `wayne`（状态活跃，角色 region-admin）。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 成员」
2. 查看默认 Tab 及页面标题
3. 查看列表表头字段
4. 查看列表数据行内容

#### 预期结果
- 页面成功加载「Region 成员」页面
- 默认展示「用户」Tab 且处于激活状态
- 列表表头包含复选框、名称、状态、角色、最近登录时间、操作
- 数据行正确展示用户信息
  1. 头像、用户名、邮箱正确显示
  2. 状态显示为活跃及绿色状态点
  3. 角色显示为 region-admin
  4. 最近登录时间格式正确
  5. 操作列展示 ⋮ 更多按钮

## 模块二：用户 Tab - 邀请成员

### tc-P0-功能测试：邀请单个用户加入 Region

#### 前置条件 (pc)
- 已使用 platform-admin 或 region-admin 角色登录。
- 平台中存在未被邀请至当前 Region 的用户 `new_user`，角色可选。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 成员」→「用户」Tab
2. 点击顶部「邀请」按钮
3. 在弹窗搜索框输入 `new_user`
4. 在可邀请用户列表中找到 `new_user`
5. 点击用户项右侧的「+」按钮
6. 在角色选项中选择 `region-viewer`
7. 查看已选用户展示
8. 点击「确定」按钮
9. 查看用户列表

#### 预期结果
- 页面成功进入「用户」Tab
- 点击「邀请」后弹出「邀请成员」弹窗
- 搜索后展示匹配的可邀请用户
- 可邀请用户列表中存在 `new_user`
- 点击「+」后展开角色选择：region-viewer、region-regular、region-admin，并展示角色说明
- 在角色选项中成功选择 `region-viewer`
- 已选用户显示「已分配角色」及对勾标识
- 弹窗关闭，页面顶部或列表中出现成功提示
- 用户列表刷新并展示新增用户 `new_user`，角色为 region-viewer

- 用户列表刷新，`user_a`、`user_b`、`user_c` 均加入 Region 且角色正确
