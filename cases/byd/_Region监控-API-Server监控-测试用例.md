# Region 监控 - API Server 监控 - 测试用例

> 本用例针对 Region 监控下「API Server 监控」Tab 页签进行细化，覆盖请求延迟、每秒请求数图表加载、图例按动词区分、集群/时间切换、图例筛选、异常/空态及国际化/权限等场景。

---

## 模块一：页面加载与图表展示

### tc-P0-功能测试：正常进入 API Server 监控页签，请求延迟和每秒请求数图表加载成功

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且 API Server 有实际请求数据。
- 监控组件（Prometheus/Whizard）正常运行。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「API Server 监控」
2. 等待页面全部图表加载完成
3. 查看请求延迟(ms)图表
4. 查看每秒请求数(次/秒)图表
5. 对比各图表数据与后端监控接口返回数据

#### 预期结果
- API Server 监控页签加载成功，无白屏、无长时间空白
- 页面顶部集群选择器和时间范围选择器默认显示正确，图表布局整齐，无错位、重叠或截断
- 请求延迟(ms)图表正常渲染，按 CREATE/DELETE/GET/POST/PATCH/PUT/UPDATE/LIST 等动词区分曲线
- 每秒请求数(次/秒)图表正常渲染，标题、单位、坐标轴、图例完整显示
- 各图表数据与后端接口返回一致，无 `--` 或空值（除非无数据）

### tc-P1-功能测试：API Server 监控图表单位与标题正确

#### 前置条件 (pc)
- 已进入 Region 监控「API Server 监控」页签。
- 页面已有监控数据。

#### 测试步骤
1. 查看请求延迟图表标题与 Y 轴单位
2. 查看每秒请求数图表标题与 Y 轴单位
3. 查看各请求动词对应的图例文案
4. 对比各图表展示与设计稿或产品说明

#### 预期结果
- 请求延迟图表标题为「请求延迟」或「API Server 请求延迟」，单位为 `ms`
- 每秒请求数图表标题为「每秒请求数」或「API Server 每秒请求数」，单位为 `次/秒`

- 各图表展示与设计稿或产品说明一致，图例颜色区分明显

## 模块二：图例与请求动词筛选

### tc-P0-功能测试：图例按请求动词正确区分颜色

#### 前置条件 (pc)
- 已进入 Region 监控「API Server 监控」页签。
- 页面图表已有按动词拆分的数据。

#### 测试步骤
1. 查看请求延迟图表图例
2. 查看每秒请求数图表图例
3. 记录各请求动词对应的颜色
4. 对比图例颜色与曲线颜色是否一致
5. 切换集群或时间范围，观察图例颜色是否保持一致

#### 预期结果
- 请求延迟图表中，各请求动词（CREATE/DELETE/GET/POST/PATCH/PUT/UPDATE/LIST 等）均有独立图例
- 每秒请求数图表中，各请求动词（CREATE/DELETE/GET/POST/PATCH/PUT/UPDATE/LIST 等）均有独立图例
- 同一动词在请求延迟和每秒请求数图表中的颜色保持一致，不同动词间颜色区分明显
- 图例颜色与对应曲线颜色一致，无错位
- 切换集群或时间范围后，图例颜色不变，数据曲线按新条件正确刷新

#### 预期结果

  1. 被隐藏图例样式变为灰色或置灰状态
  2. 图表其他部分正常显示
- 对应曲线重新显示
- 所有请求动词曲线均可单独隐藏/显示，操作过程中图表无报错或白屏
- 同时隐藏多个动词并依次恢复后，图表坐标轴与其他曲线自适应调整，展示正常

## 模块三：集群与时间范围切换

### tc-P0-功能测试：切换集群后 API Server 图表数据刷新

#### 前置条件 (pc)
- 平台下已存在包含多个成员集群的 Region。
- 各集群 API Server 请求数据存在明显差异。
- 已进入 Region 监控「API Server 监控」页签。

#### 测试步骤
1. 记录当前集群选择器和各图表数据
2. 在页面顶部集群选择器中切换至另一个成员集群
3. 观察请求延迟和每秒请求数图表加载与刷新过程
4. 记录切换后各图表数据并与切换前对比

#### 预期结果
- 当前集群选择器和各图表数据正常展示，可作为对比基线
- 页面给出加载提示，请求延迟和每秒请求数图表按新集群重新加载
- 各请求动词曲线数据均刷新为所选集群的 API Server 监控数据
- 切换后的数据与所选集群实际 API Server 请求数据一致，无旧数据残留、无数据错乱；切换过程中页面不白屏、不报错

## 模块五：异常与空态

### tc-P1-功能测试：监控数据为空或接口异常时的处理

#### 前置条件 (pc)
- 当前 Region/集群下 API Server 无请求监控数据。
- 或通过 Mock 模拟 API Server 监控接口返回空数据或异常。
- 已进入 Region 监控「API Server 监控」页签。

#### 测试步骤
1. 进入 API Server 监控页签
2. 查看各图表空态展示
3. 模拟请求延迟接口返回空数据，刷新页面
4. 模拟每秒请求数接口 500 错误，刷新页面
5. 模拟全部接口异常，刷新页面
6. 恢复接口后刷新页面

#### 预期结果
- 页面正常加载
- 无监控数据的图表展示空态提示（如「暂无数据」或空折线图），空态提示文案正确，符合当前语言环境
- 请求延迟图表展示空态提示或占位，每秒请求数图表正常加载
- 每秒请求数图表展示错误提示或占位，请求延迟图表正常加载
- 页面展示明确的错误提示或重试入口，不白屏
  1. 接口超时时，展示加载超时提示，不长时间卡死
  2. 错误提示文案正确，符合当前语言环境
- 页面可正常刷新并展示数据

#### 前置条件 (pc)
- 已使用非 platform-admin 角色登录（如 platform-regular、自定义只读角色）。
- 平台下已存在 Region 及成员集群。

#### 测试步骤
1. 使用无权限用户登录平台
2. 直接访问 API Server 监控页 URL
3. 尝试调用 API Server 监控相关 API 接口
4. 切换为 platform-admin 角色，再次访问并对比

#### 预期结果
- 无权限用户无法通过 UI 进入 API Server 监控页，或页面提示无权限
- 页面跳转至无权限提示或 403 页面
- 接口返回 403 无权限
- platform-admin 角色可正常访问并查看全部数据；数据权限隔离正确，不会越权展示其他 Region/集群数据
