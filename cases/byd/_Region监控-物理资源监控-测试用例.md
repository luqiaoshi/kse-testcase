# Region 监控 - 物理资源监控 - 测试用例

> 本用例针对 Region 监控下「物理资源监控」Tab 页签进行细化，覆盖页面加载、集群与时间范围切换、图表交互、异常/空态及国际化/权限等场景。

---

## 模块一：页面加载与图表展示

### tc-P0-功能测试：正常进入物理资源监控页签，所有图表加载成功

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且集群有实际物理资源监控数据。
- 监控组件（Prometheus/Whizard）正常运行。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「物理资源监控」
2. 等待页面全部图表加载完成
3. 依次查看 CPU 用量(%)、内存用量(%)、CPU 平均负载（1/5/15 分钟）、磁盘用量(GB)、inode 用量(%)、磁盘吞吐(MB/s)、IOPS、网络流量(Mbps)、容器组状态等图表
4. 对比各图表数据与后端监控接口返回数据

#### 预期结果
- 物理资源监控页签正确加载，无白屏、无长时间空白
- 页面顶部集群选择器和时间范围选择器默认显示正确
- 所有图表均正常渲染，标题、单位、坐标轴、图例完整显示
  1. CPU 用量图表正常显示
  2. 内存用量图表正常显示
  3. CPU 平均负载图表正常显示
  4. 磁盘用量图表正常显示
  5. inode 用量图表正常显示
  6. 磁盘吞吐图表正常显示
  7. IOPS 图表正常显示
  8. 网络流量图表正常显示
  9. 容器组状态图表正常显示
- 各图表数据与后端接口返回一致，无 `--` 或空值（除非无监控数据）

### tc-P1-功能测试：各物理资源监控图表单位与标题正确

#### 前置条件 (pc)
- 已进入 Region 监控「物理资源监控」页签。
- 页面已有监控数据。

#### 测试步骤
1. 查看 CPU 用量图表标题与 Y 轴单位
2. 查看内存用量图表标题与 Y 轴单位
3. 查看 CPU 平均负载图表标题与 1/5/15 分钟曲线说明
4. 查看磁盘用量、inode 用量、磁盘吞吐、IOPS、网络流量、容器组状态等图表标题与单位
5. 对比各图表展示与设计稿或产品说明

#### 预期结果
- CPU 用量图表标题为「CPU 用量」，单位为 `%`
- 内存用量图表标题为「内存用量」，单位为 `%`
- CPU 平均负载图表展示 1 分钟、5 分钟、15 分钟三条曲线，单位/说明正确
- 磁盘用量、inode 用量、磁盘吞吐、网络流量等图表单位正确
  1. 磁盘用量单位为 `GB`
  2. inode 用量单位为 `%`
  3. 磁盘吞吐单位为 `MB/s`
  4. 网络流量单位为 `Mbps`
- 各图表展示与设计稿或产品说明一致，IOPS、容器组状态图表标题与单位正确
  1. 所有图表图例文案与对应指标一致

## 模块二：集群选择器与时间范围切换

### tc-P0-功能测试：切换集群后，所有图表数据刷新为对应集群

#### 前置条件 (pc)
- 平台下已存在包含多个成员集群的 Region。
- 各集群在 CPU、内存、磁盘、网络、容器组等监控数据上存在明显差异。
- 已进入 Region 监控「物理资源监控」页签。

#### 测试步骤
1. 记录当前集群选择器和各图表数据
2. 在页面顶部集群选择器中切换至另一个成员集群
3. 观察各图表加载与刷新过程
4. 记录切换后各图表数据并与切换前对比

#### 预期结果
- 页面显示加载提示，所有图表按所选集群重新加载
- CPU、内存、CPU 平均负载、磁盘、inode、磁盘吞吐、IOPS、网络流量、容器组状态等图表数据均为所选集群数据
- 切换后数据与所选集群实际监控数据一致
- 切换过程无旧数据残留、无数据错乱，页面不白屏、不报错

### tc-P1-功能测试：切换时间范围后，图表数据刷新

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且有实际监控数据。
- 已进入 Region 监控「物理资源监控」页签。

#### 测试步骤
1. 记录当前时间范围选择器和各图表数据
2. 点击时间范围选择器，切换为「最近 2 小时」
3. 观察各图表加载与刷新过程
4. 切换为其他时间范围（如最近 6 小时、12 小时、24 小时、7 天等，以实际支持为准）

#### 预期结果
- 所有图表按所选时间范围重新查询并刷新
- 图表 X 轴时间刻度随时间范围变化正确调整
- 各图表展示所选时间范围内的监控数据，数据点分布合理
- 切换过程页面给出加载提示，无旧数据残留、无数据错乱

#### 测试步骤
1. 进入物理资源监控页签
2. 查看各图表空态展示

4. 模拟全部图表接口返回空数据，刷新页面

#### 预期结果

2. 查看页面标题、图表标题、图例、单位、提示信息等文案

### tc-P2-安全性测试：无权限用户访问物理资源监控控制

#### 前置条件 (pc)
- 已使用非 platform-admin 角色登录（如 platform-regular、自定义只读角色）。
- 平台下已存在 Region 及成员集群。

#### 测试步骤
1. 使用无权限用户登录平台
2. 直接访问物理资源监控页 URL
3. 尝试调用物理资源监控相关 API 接口
4. 切换为 platform-admin 角色，再次访问并对比

#### 预期结果
- 无权限用户无法通过 UI 进入物理资源监控页，或页面提示无权限
- 直接访问 URL 时，页面跳转至无权限提示或 403 页面
- 通过接口直接调用物理资源监控相关 API 时，接口返回 403 无权限
- platform-admin 角色可正常访问并查看全部数据，数据权限隔离正确，不会越权展示其他 Region/集群数据
