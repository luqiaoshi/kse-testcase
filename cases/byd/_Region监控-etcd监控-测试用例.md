# Region 监控 - etcd 监控 - 测试用例

> 本用例针对 Region 监控下「etcd 监控」Tab 页签进行细化，覆盖服务状态、节点信息、图表加载、集群/时间切换、异常/空态及国际化/权限等场景。

---

## 模块一：页面加载与服务状态

### tc-P0-功能测试：正常进入 etcd 监控页签，服务状态与图表加载成功

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且集群使用外部 etcd 或内部 etcd（以实际部署为准）。
- etcd 服务正常运行，有实际监控数据。
- 监控组件（Prometheus/Whizard）正常运行。
- 已使用 platform-admin 角色登录平台。

#### 测试步骤
1. 进入「平台管理」→「Region 管理」→「Region 监控」→「etcd 监控」
2. 等待页面服务状态区域加载完成
3. 查看各外部 etcd 节点信息：节点 IP、是否有 Leader、1 小时内 Leader 变更次数、leader 标签
4. 依次查看库大小、客户端流量、gRPC 流式消息、WAL 日志同步时间、库同步时间、Raft 提议等图表
5. 对比服务状态与图表数据和后端接口返回

#### 预期结果
- 页面正常加载，无白屏、无长时间空白
- 页面顶部集群选择器和时间范围选择器默认显示正确，服务状态区域数据正常展示
- 服务状态区域展示各 etcd 节点信息，字段无缺失
  1. 节点 IP 展示正确
  2. 是否有 Leader 展示正确
  3. 1 小时内 Leader 变更次数展示正确
  4. leader 标签展示正确
- 各图表正常渲染，标题、单位、坐标轴、图例完整显示
  1. 库大小图表正常渲染，标题、单位、坐标轴、图例完整显示
  2. 客户端流量图表正常渲染，标题、单位、坐标轴、图例完整显示
  3. gRPC 流式消息图表正常渲染，标题、单位、坐标轴、图例完整显示
  4. WAL 日志同步时间图表正常渲染，标题、单位、坐标轴、图例完整显示
  5. 库同步时间图表正常渲染，标题、单位、坐标轴、图例完整显示
  6. Raft 提议图表正常渲染，标题、单位、坐标轴、图例完整显示
- 服务状态与图表数据与后端接口返回一致，无 `--` 或空值（除非无数据）

### tc-P1-功能测试：etcd 服务状态区域字段完整

#### 前置条件 (pc)
- 已进入 Region 监控「etcd 监控」页签。
- 服务状态区域有节点数据。

#### 测试步骤
1. 查看服务状态区域表头/卡片字段
2. 查看各 etcd 节点的节点 IP、是否有 Leader、1 小时内 Leader 变更次数、leader 标签
3. 对比展示字段与设计说明

#### 预期结果
- 服务状态区域表头/卡片字段完整，布局整齐，无缺失或错位
- 各 etcd 节点信息字段展示正确
  1. 节点 IP 字段展示，且 IP 格式正确
  2. 「是否有 Leader」字段正确展示是/否或 true/false
  3. 「1 小时内 Leader 变更次数」字段正确展示数值
  4. leader 节点有明确的 leader 标签标识
- 展示字段与设计说明一致，无缺失或错位

## 模块二：节点信息与 Leader 校验

### tc-P0-功能测试：各 etcd 节点信息显示正确

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群。
- 已准备当前集群下各 etcd 节点的真实信息（IP、Leader 状态、变更次数等）。
- 已进入 Region 监控「etcd 监控」页签。

#### 测试步骤
1. 查看服务状态区域各 etcd 节点列表
2. 逐行核对节点 IP 与后端 etcd 节点信息
3. 核对「是否有 Leader」状态
4. 核对「1 小时内 Leader 变更次数」
5. 切换不同集群，重复核对

#### 预期结果
- 服务状态区域展示当前集群各 etcd 节点列表，信息无缺失
- 各节点 IP 与后端 etcd 实际节点 IP 一致
- 「是否有 Leader」状态与后端返回一致
- 「1 小时内 Leader 变更次数」与后端返回一致
- 切换集群后，服务状态区域按所选集群 etcd 节点信息刷新，无旧数据残留、无数据错乱

### tc-P1-功能测试：leader 节点标签显示正确

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且 etcd 集群中存在 leader 节点。
- 已进入 Region 监控「etcd 监控」页签。

#### 测试步骤
1. 查看服务状态区域各节点
2. 定位标注为 leader 的节点
3. 对比后端接口返回的 leader 节点信息
4. 在 etcd 实际集群中确认 leader 节点（如通过 etcdctl）
5. 切换时间范围，观察 leader 标签变化

#### 预期结果
- 服务状态区域展示各 etcd 节点信息，无缺失或错位
- leader 节点正确显示 leader 标签，样式/颜色与从节点区分明显
- leader 标签对应的节点与后端接口返回的 leader 节点一致
- leader 标签与实际 etcd 集群 leader 节点一致
- 切换时间范围不影响 leader 标签展示；leader 变更后，标签状态及时更新

### tc-P2-功能测试：Leader 变更次数统计正确

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群。
- 已触发或已知 1 小时内发生过 Leader 变更。
- 已进入 Region 监控「etcd 监控」页签。

#### 测试步骤
1. 查看服务状态区域「1 小时内 Leader 变更次数」列
2. 记录各节点变更次数
3. 对比后端接口返回的 Leader 变更统计数据
4. 模拟 Leader 变更（如在测试环境触发切换），刷新页面查看变化

#### 预期结果
- 服务状态区域展示「1 小时内 Leader 变更次数」列，字段无缺失
- 各节点变更次数记录正确，为整数，显示格式正确
- 「1 小时内 Leader 变更次数」与后端接口返回一致
- Leader 变更后，变更次数在合理时间内刷新并增加；无变更时显示为 0

### tc-P0-功能测试：切换集群后，etcd 状态和图表数据刷新

#### 前置条件 (pc)
- 平台下已存在包含多个成员集群的 Region。
- 各集群 etcd 节点信息与监控数据存在明显差异。
- 已进入 Region 监控「etcd 监控」页签。

#### 测试步骤
1. 记录当前集群选择器、服务状态区域和各图表数据
2. 在页面顶部集群选择器中切换至另一个成员集群
3. 观察服务状态区域和图表加载与刷新过程
4. 记录切换后数据并与切换前对比

#### 预期结果
- 切换前，当前集群选择器、服务状态区域和各图表数据正常展示
- 切换集群后，页面给出加载提示
- 服务状态区域按新集群 etcd 节点信息刷新，各图表数据均刷新为所选集群的数据
  1. 节点 IP、Leader 状态、变更次数、leader 标签均正确
  2. 库大小、客户端流量、gRPC 流式消息、WAL 日志同步时间、库同步时间、Raft 提议等图表数据均刷新为所选集群的数据
- 切换后的数据与所选集群实际 etcd 监控数据一致，切换过程中页面不白屏、不报错；无旧数据残留、无数据错乱

### tc-P1-功能测试：切换时间范围后 etcd 数据刷新

#### 前置条件 (pc)
- 平台下已存在 Region 及成员集群，且有实际 etcd 监控数据。
- 已进入 Region 监控「etcd 监控」页签。

#### 测试步骤
1. 记录当前时间范围选择器和服务状态、各图表数据
2. 点击时间范围选择器，切换为「最近 2 小时」
3. 观察各图表加载与刷新过程

- 切换为「最近 2 小时」后，所有图表按新时间范围重新查询并刷新
- 各图表加载与刷新过程正常
  1. 图表 X 轴时间刻度随时间范围变化正确调整
  2. 各图表展示所选时间范围内的 etcd 监控数据
- 切换为其他时间范围后，服务状态区域（Leader、变更次数等）保持实时/准实时展示，切换过程中页面给出加载提示，无旧数据残留、无数据错乱

### tc-P1-功能测试：etcd 监控数据为空时图表空态展示

#### 前置条件 (pc)
- 当前 Region/集群下 etcd 无监控数据。

2. 查看各图表空态展示

- 页面正常加载，无白屏、无报错

  1. 空态提示文案正确，符合当前语言环境
- 单个图表空数据时，仅该图表展示空态，其他图表正常展示
- 全部图表空数据时，页面整体布局正常，无白屏、无报错

### tc-P2-安全性测试：无权限用户访问 etcd 监控控制

#### 前置条件 (pc)
- 已使用非 platform-admin 角色登录（如 platform-regular、自定义只读角色）。
- 平台下已存在 Region 及成员集群。

#### 测试步骤
1. 使用无权限用户登录平台
2. 直接访问 etcd 监控页 URL
3. 尝试调用 etcd 监控相关 API 接口
4. 切换为 platform-admin 角色，再次访问并对比

#### 预期结果
- 无权限用户无法通过 UI 进入 etcd 监控页，或页面提示无权限
- 直接访问 URL 时，页面跳转至无权限提示或 403 页面
- 通过接口直接调用 etcd 监控相关 API 时，接口返回 403 无权限
- 切换为 platform-admin 角色后可正常访问并查看全部数据；数据权限隔离正确，不会越权展示其他 Region/集群数据
