## 模块 1：License 增量导入

### 核心规则
- License 支持**全量许可**和**增量许可**两种类型
- **新环境首次导入**必须是全量许可
- 增量许可不允许在没有全量许可的环境中导入
- 后续扩容可导入增量许可
- 同 license_id 重复导入时，**幂等覆盖**处理
- **新全量许可导入**后，旧全量许可状态固定变为"已作废"
- **新全量许可导入**后，旧增量许可仍然有效
- **有效授权规模** = 当前有效期内授权总量
- **增量许可到期**后，自动从有效授权总量中扣减
- 增量许可到期导致授权不足时：**提示 + 限制使用**
- **已过期/已作废记录**长期保留，默认列表展示

### 时间口径
- 订阅截止时间：取最早到期授权（平台级口径）
- 概览页只展示平台级订阅截止时间，不展示不同组件的订阅截止时间
- 到期判断统一使用 UTC

### 页面展示影响
- 授权概览展示：授权类型、已使用、授权规模、订阅生效时间、订阅截止时间
- 订阅截止时间增加 tooltip：*"若存在多批次导入许可，截止时间按最早到期许可核定"*
- 软件许可导入记录展示：许可证ID、授权类型、许可证类型、许可证状态、授权数量、导入时间、生效时间、截止时间、剩余有效期
- 许可证状态：已生效 / 已过期 / 已作废
- 许可剩余有效期 < 30 天时，文字变红

### 待测场景
| 场景 | 说明 |
|------|------|
| 首次导入全量许可 | 正常流程 |
| 全量许可后导入增量许可 | 扩容场景 |
| 无全量许可直接导入增量 | 异常拦截 |
| 重复导入同 license_id | 幂等覆盖 |
| 新全量覆盖旧全量 | 旧全量→已作废，旧增量仍有效 |
| 增量许可到期 | 自动扣减，不足时限制 |
| 多批次增量到期时间不同 | 最早到期为准 |
| 已过期记录展示 | 默认展示，长期保留 |

## 模块 2：软件 License 与维保 License 拆分

### 字段定义

**软件许可字段：**
| 字段 | 说明 |
|------|------|
| 组件名称 | - |
| 许可证ID | 取 ks-core 的 ID |
| 授权数量 | - |
| 授权状态 | 已生效 / 已过期 / 已作废 |

**维保许可字段：**
| 字段 | 说明 |
|------|------|
| 组件名称 | - |
| 许可证ID | 取 ks-core 的 ID |
| 授权数量 | 可设置为 0 |
| 授权状态 | 已生效 / 已到期 |
| 维保生效时间 | - |
| 维保到期时间 | - |

### 核心规则
- 软件 License 和维保 License **独立申请、导入、展示**
- 两者都以 ks-core 作为主组件归属
- 维保 License 不参与数量控制
- 维保 License 不需要与软件 License 授权数量一致
- 多批次维保续约时，**维保到期时间按最终时间计算**
- 已过期记录长期保留，默认列表展示

### 维保到期策略
- 维保 License 支持一个**特殊标识**（后端字段），控制到期后是否限制平台功能
- 特殊标识由 kscloud 签发
- **默认值**：不限制平台功能（仅提示）
- 如果开启该标识：维保到期后限制平台使用，完全等同订阅到期策略

### 维保提醒
- **T-90 天**：每周提醒一次，点击关闭一周不再显示
- **T-30 天**：每天提醒，点击关闭一天不再显示
- 采用弱提醒，允许关掉

### 页面展示影响
- 永久许可概览：授权类型、已使用、授权规模、维保生效时间、维保到期时间
- 维保到期时间 tooltip：*"若存在多批次续约维保，截止时间按最终维保到期时间核定"*
- 维保许可导入记录：许可证ID、许可证状态、授权数量、导入时间、维保生效时间、维保到期时间、查看详情

### 待测场景
| 场景 | 说明 |
|------|------|
| 导入软件许可 | 正常流程 |
| 导入维保许可 | 正常流程 |
| 维保授权数量为 0 | 允许 |
| 维保数量与软件数量不一致 | 各自独立 |
| 维保到期（无特殊标识） | 仅提示，不限制 |
| 维保到期（有特殊标识） | 限制平台功能 |
| 多批次维保续约 | 到期时间按最终计算 |
| 维保到期提醒（90天） | 每周提醒+关闭 |
| 维保到期提醒（30天） | 每天提醒+关闭 |

## 模块 3：分级授权与扩展组件权限

### 核心规则
- 扩展组件增加**许可等级标识**：社区版 / 企业版 / 旗舰版
- License 中包含两部分授权信息：**许可等级** + **额外扩展组件 ID**
- 许可等级用于判断默认可用扩展范围
- 额外扩展组件 ID 在默认等级范围外补充授权
- 扩展组件等级映射由 License 服务维护

### 兼容期判断
- 新旧 License 共存时：**许可等级授权 OR 扩展组件 ID 授权**，任一满足即算有权限
- 新版本 License 按许可等级动态判断
- 旧版本 License 仍按扩展组件 ID 判断

### 增量许可下的扩展组件规则
- 增量许可默认继承环境中全量许可的扩展组件授权范围
- 如果授权范围发生变化，需要重新申请全量许可
- 增量许可只校验增加的 vCPU 数量，不重新定义扩展组件授权范围

### 页面展示影响
- 授权概览第三行：`扩展组件（35个）：KubeSphere 网关、wizardTelemetry 事件……` 支持展开/收起
- 许可详情中展示扩展组件授权范围

### 待测场景
| 场景 | 说明 |
|------|------|
| 新 License 按等级授权 | 企业版可访问企业版扩展 |
| 旧 License 按 ID 授权 | 传统方式 |
| 新旧 License 共存 | OR 逻辑判断 |
| 增量许可继承扩展范围 | 与全量一致 |
| 扩展组件展示 | 展开/收起 |
| 许可详情-扩展组件列表 | 数据正确性 |

## 模块 4：顶部 License Banner 文案

### state 与文案映射

| state | violation.type | 管理员文案 | 非管理员文案 |
|-------|---------------|-----------|-------------|
| unauthorized | Empty license | KubeSphere 未激活。请激活 KubeSphere。 | KubeSphere 未激活。请联系平台管理员激活 KubeSphere。 |
| unauthorized | Time not start | 许可证生效时间未开始，请输入当前有效的许可证。 | 许可证生效时间未开始，请联系平台管理员输入当前有效的许可证。 |
| unauthorized | Cluster not match | 许可证集群 ID 不匹配，请输入正确的许可证。 | 许可证集群 ID 不匹配，请联系平台管理员输入正确的许可证。 |
| unauthorized | 其他/无值 | KubeSphere 未激活。请激活 KubeSphere。 | KubeSphere 未激活。请联系平台管理员激活 KubeSphere。 |
| expired | - | 许可证已过期。请更新许可证。 | 许可证已过期。请联系平台管理员更新许可证。 |
| overLimit | - | 您当前的配额已达到上限。如需继续使用，请联系我们的客服团队了解升级方案。 | （同上，不区分角色） |
| authorized(即将到期) | - | 订阅服务将于 X 天后过期。请更新许可证。 | 订阅服务将于 X 天后过期。请联系平台管理员更新许可证。 |
| authorized(正常) | - | 不显示任何提示 | 不显示任何提示 |

### 待测场景
| 场景 | 说明 |
|------|------|
| 无 License | unauthorized + Empty license |
| License 未到生效时间 | unauthorized + Time not start |
| License 集群不匹配 | unauthorized + Cluster not match |
| License 已过期 | expired |
| 超过用量 | overLimit（不分角色） |
| License 即将到期 | 7天/1天等边界 |
| License 正常 | 无提示 |
| 平台管理员 vs 非管理员 | 所有场景分别验证 |

## 模块 7：生成授权许可凭证

### 功能描述
- 后端提供接口，Telemetry 生成的环境信息数据发送给前端
- 前端将数据复制到剪贴板
- 该数据作为申请许可的必要凭证

### 待测场景
| 场景 | 说明 |
|------|------|
| 点击"生成授权许可凭证" | 按钮存在且可点击 |
| 数据复制到剪贴板 | 成功复制 |
| 复制内容格式正确 | JSON 或其他格式 |
| 无数据时按钮状态 | 不可点击或提示 |
| 权限校验 | 非管理员不可操作 |
