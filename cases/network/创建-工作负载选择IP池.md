## tc-P1-功能测试：创建工作负载时选择 IPPool，成功

### 前置条件 (pc)
- 存在 IPv4/IPv6/DualStack 类型的 IPPool

### 测试步骤 & 预期结果
1. 进入【项目】-【工作负载】，创建部署 → 进入创建工作负载流程
2. 填写基本信息、容器组设置、存储设置，点击：下一步 → 跳转至高级设置
3. 找到添加容器组IP池模块 → 显示 IPPool 选择列表
4. 查看列表字段（名称、类型、网段、已用IP地址、IP池块大小） → 各字段显示正确
5. 按名称搜索 → 列表实时筛选
6. 按类型筛选 → 列表过滤正确
7. 勾选一个 IPv4 IPPool → 回填区域显示：已选择1个IP池，IPv4：IP池名称
8. 继续勾选一个 IPv6 IPPool → 回填区域更新：已选择2个IP池，IPv4：IP池名称，IPv6：IP池名称
9. 取消勾选 → 回填区域同步更新
10. 确认选择，点击：创建 → 工作负载创建成功，容器组分配到对应 IP 池的 IP 地址

## tc-P1-功能测试：编辑工作负载时更换 IPPool，成功

### 前置条件 (pc)
- 存在已绑定 IPPool 的工作负载
- 存在其他可选的 IPPool

### 测试步骤 & 预期结果
1. 进入工作负载详情页，操作：编辑设置 → 进入编辑页面
2. 找到 IPPool 选择模块 → 已勾选的 IPPool 正确回填
3. 回填区域显示：已选择x个IP池，IPv4：IP池名称，IPv6：IP池名称
4. 取消原有 IPPool，重新勾选新的 IPPool → 回填区域同步更新
5. 按名称搜索 → 列表实时筛选
6. 按类型筛选 → 列表过滤正确
7. 保存设置 → 编辑成功，容器组 IP 分配更新
