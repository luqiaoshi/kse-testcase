## tc-P1-功能测试：创建 IPv4 类型 IPPool，成功

### 前置条件 (pc)
- 集群已开启 IPv6 双栈模式，CNI 插件支持 IPv6

4. 步骤2预览表格显示（IP池名称、状态、类型、网段、IP池块大小、Overlay模式） → 预览数据正确
5. 可编辑已创建的 IP 池名称，启用/禁用 → 编辑功能正常
6. 点击：创建 → 创建成功，列表显示新增的 IPv4 IPPool

## tc-P3-功能测试：创建 IPPool，字段校验正确

### 前置条件 (pc)
- 集群运行正常

### 测试步骤 & 预期结果
1. 进入【集群】-【容器组IP池】，点击：创建 → 弹出创建页
2. 不输入 CIDR → 不可下一步，提示必填
3. 输入非法 IPv4 CIDR 格式（如 42.0.0/16、42.0.0.0.0/16、abc） → 前端校验拦截，提示格式错误
4. 输入非法 IPv6 CIDR 格式（如 fd00:1234::/64xxx、fd00:::/64、abc） → 前端校验拦截，提示格式错误
5. IPv4 CIDR 超出私有网段范围（如 99.0.0.0/8） → 按需求决定是否允许，若不允许则提示
6. IPv6 地址格式输入正确但子网掩码不合法（如 /129） → 前端校验拦截，提示子网掩码范围错误
7. 数量输入 0 → 前端校验拦截，不可保存
8. 数量输入负数（如 -1） → 前端校验拦截，不可保存
9. 数量输入小数（如 5.5） → 前端校验拦截，不可保存
10. 数量输入超过 10（如 11） → 前端校验拦截，提示最多创建 10 个
11. 数量输入 10 → 可正常下一步
12. 类型切换（IPv4→IPv6→DualStack→IPv4） → 配置项正确切换
13. DualStack 类型下，仅输入 IPv4 CIDR 不输入 IPv6 CIDR → 不可下一步，提示 IPv6 CIDR 必填
14. DualStack 类型下，IPv4 CIDR 与 IPv6 CIDR 网段冲突 → 前端校验拦截，提示冲突

## tc-P3-功能测试：创建 IPPool 步骤交互

### 前置条件 (pc)
- 集群运行正常

### 测试步骤 & 预期结果
1. 进入【集群】-【容器组IP池】，点击：创建 → 弹出创建页，默认步骤1
2. 步骤1未填写必填项，点击：下一步 → 不可跳转，提示必填
3. 填写正确信息，点击：下一步 → 跳转至步骤2预览
4. 步骤2页面点击：上一步 → 返回步骤1，已填写信息保留
5. 修改步骤1信息后，再次点击：下一步 → 步骤2预览数据同步更新
6. 步骤2点击：取消 → 创建流程关闭，列表无新增数据
7. 步骤1点击：取消 → 创建流程关闭，列表无新增数据
